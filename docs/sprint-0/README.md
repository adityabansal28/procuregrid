## Working Assumptions

- Sprint 0 is for planning and alignment, not full feature delivery.
- MVP scope remains procurement-only and should not drift into ERP breadth.
- Escrow is represented as a tracked workflow state in the MVP and must be backed by a regulated payment partner later.
- The initial user groups are buyers, suppliers, and platform admins.
- Vertical focus is still open, so the architecture remains category-agnostic even though the marketing language is manufacturing-first.

## Exit Criteria

Sprint 0 is complete when the team agrees on:

- What the MVP includes and excludes
- The first user journeys to build
- The system boundaries and service ownership
- The initial relational schema and state transitions
- The starting backlog for Sprint 1
